# Zamblak Docs Guard

**When to use:** When adding or updating documentation.
**What it guards:** Ensures `docs/` structure is consistent and acts as the source of truth alongside code.
**Required checks:** Verify changes against `AGENTS.md` rules.
**Forbidden actions:** Bypassing documentation for core architectural changes.
**PASS/HOLD expectations:** Documentation is updated, accurate, and properly formatted.
